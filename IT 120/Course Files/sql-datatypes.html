Typed data

SQL attributes are “typed”: that is, the attribute must be declared not only with a name but also the kind of data that can be stored in the attribute for instances of the entity.

The data type conveys semantics about the value that may be stored:

  • length
  • sorting behavior

For numeric types:

  • size (implies minimum and maximum values that can be stored)
  • fractional support (e.g. integer, floating point, or fixed point)

~ 22 types specified by the SQL standard; another 20 or so are provided by PostgreSQL.

Sorting behavior examples

Numerics

Compare the sorting of numeric literals 22, 5, and 1:

abc=> SELECT * FROM
    (VALUES (22), (5), (1))
    AS x ORDER BY x;
 column1 
---------
       1
       5
      22
(3 rows)

With the same literals as strings:

abc=> SELECT * FROM (
    VALUES ('22'), ('5'), ('1'))
    AS x ORDER BY x;
 column1 
---------
 1
 22
 5
(3 rows)

Strings

Strings sort according to the server’s LOCALE/collation configuration.

On a Mac, installed with brew, using the default “C” locale:

abc=> SELECT * FROM (
    VALUES ('ABC'), ('SQL'), ('ignores case'), ('alphabetic'))
    AS x ORDER BY x;
   column1    
--------------
 ABC
 SQL
 alphabetic
 ignores case
(4 rows)

On Windows, using UTF-8 encoding and US-English.1252 collation:

abc=> SELECT * FROM (
     VALUES ('ABC'), ('SQL'), ('ignores case'), ('alphabetic'))
     AS x ORDER BY x;
   column1
--------------
 ABC
 alphabetic
 ignores case
 SQL
(4 rows)

Core types

For this course, we will primarily encounter

  • VARCHAR or TEXT, for text
  • INTEGER, often for keys or counting
  • NUMERIC, for fixed-point needs
  • DATE, for dates
  • TIMESTAMP, for recording events (msec precision)

Text

CHAR(n) is a fixed-width field. Text that does not fill the field is padded. Codes–like UPC–are the sort of application suited for a CHAR.

VARCHAR(n) is a variable-width field: it can be any length up to the specified n. PostgreSQL allows the parameter to be omitted, and the field is of unlimited length.

TEXT is an unlimited length string.

Strings sort according to the LOCALE conventions (basically: alphabetical by leading characters).

Numeric

INTEGER is a 4-byte signed value.

NUMERIC(precision, scale) (also known as DECIMAL): a base-10 representation with a total of precision places, with scale of those to the right of decimal point representing the fractional component. NUMERIC(5,3) can represent 99.001, but 100 will overflow and 0.0493 will get truncated. Useful in business/finance.

REAL/DOUBLE are floating-point numbers.

Numbers sort as you would expect.

Date and datetime

These represent dates (to the day), or moments on a specific day (date with time). (There are also times and time intervals for computing dates in the future; ages; things like that.)

Dates compare with the past being less than the future.

Literals are enclosed in single quotes, e.g. ‘January 8, 99 BC’ or ‘1997-12-17 07:37:16-08’

PostgreSQL can parse various dates and timezone formats, and you can set database modes to have a default. This may be useful if you are importing data. However, ISO 8601 is always the preferred date format.

ISO 8601: YYYY-MM-DD
