Working with PostgreSQL

All our work in PostgreSQL will be done via the “psql” command. This command is run within the pgAdmin app, or it can be invoked directly.

Patterns

pgAdmin

Note that pgAdmin is a “weblet”: an application that uses a web browser as an interface engine and has that talk to a local, dedicated webserver. This applet may not use the same editing commands or menuing systems you are familiar with. (And on close, it asks you “you have unedited changes; are you sure you want to leave this page?)

It is very useful if you are trying to keep track of servers running on different machines, or to open up a console quickly. It is very good for browsing tables and exploring the database.

psql with a text editor

For developing SQL, it is probably better to work with your regular editor, write code in files (and regularly commit these to SCM/git). Code can then be delivered to PostgreSQL two ways:

  • copy/paste into a psql command window
  • send entire files to the database using psql’s “-f” option

Setup: PATH for pgAdmin

The path to the psql client needs to be set for pgAdmin to work.

The menu item for this is:

  • File
  • Preferences
  • Paths
  • Binary paths

In the “PostgreSQL 15” field at the very bottom (scroll!), enter the path:

C:\Program Files\PostgreSQL\15\bin

Also: select the radio button that is “Set as default” next to this path.

Setup: PATH for programs

For working either with the command line or with VSCode, you will need a one-time step to make sure the executables are in your PATH.

On Windows, either run the following PowerShell to append to path, or do this graphically:

[System.Environment]::SetEnvironmentVariable("Path", $env:Path + ";" + "C:\Program Files\PostgreSQL\15\bin", [EnvironmentVariableTarget]::User)

(you will need to restart the PowerShell to see the effects of adding the path)

Or: search for “Edit environment variables for your account”; click “edit” and add:

C:\Program Files\PostgreSQL\15\bin
